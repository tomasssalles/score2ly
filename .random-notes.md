## LLM stack

Dependencies: **litellm** + **instructor** + **pydantic**.

- **litellm** routes requests to different providers using a single API. The model string follows the `provider/model-name` convention (e.g. `anthropic/claude-opus-4-6`, `gemini/gemini-2.5-pro`). See https://docs.litellm.ai/docs/providers.
- **instructor** wraps the litellm client and handles structured output via pydantic response models, plus retry logic that feeds schema validation errors back to the model.
- **pydantic** defines the response schemas.

Supported providers: Any for which model string + API key is sufficient. Includes:
OpenAI, Anthropic, Google Gemini, Mistral AI, DeepSeek, Groq, Cohere, Together AI, OpenRouter.

Not supported (require additional config beyond model + API key):
AWS Bedrock, Azure, local LLMs, self-hosted LiteLLM proxies. Potential future work.

Max retries (instructor): exposed as `FixSettings.max_retries`, default 2. Conservative default to avoid runaway costs from retry loops.
